Deploiements du Do-it-yourself

:::avertissement

Êtes-vous sûr de vouloir construire une nouvelle application en utilisant core ou engine ? L'offre microservice, d'OpenFn entièrement gratuite et open-source, constitue une assez bonne base dans le cas où vous voudriez étendre nos fonctionnalités existantes pour vos propres besoins.

:::

:::remarque

Nous sommes en train de migrer la documentation pour OpenFn/core et OpenFn/engine ici.

Entre-temps, consultez :

  • https://github.com/OpenFn/core
  • https://github.com/OpenFn/engine

:::

Utilisation directe de core et engine​

Par où commencer​

Les outils OpenFn core ETL sont tous open-source, et ici nous allons expliquer comment ces outils peuvent être utilisés pour effectuer des opérations ETL depuis votre ligne de commande. Vous pouvez même aller plus loin et les regrouper dans votre propre service hébergé !

Pour créer un service d'intégration comme platform ou microservice, vous devrez construire un point de terminaison REST qui permet de poster des messages JSON ou XML, renvoyant un 2XX et vérifiant le corps de ce message pour voir s'il correspond à certains critères. Si les critères correspondent à ce message, vous devez procéder à execute depuis core, en utilisant les données du message et une certaine configuration stockée comme state. Voir un exemple de state.json ci-dessous :

State.json​

{
  "data": {
    "word_count": 284,
    "last_update": "2016-11-10 13:58:47",
    "folder_name": "Civil Disobedience",
    "file_owner": "student@school.org.za",
    "file_name": "Mock Journal Article"
  },
  "configuration": {
    "host": "109.XXX.11X.2XX",
    "port": "5432",
    "database": "data-warehouse",
    "user": "postgres",
    "password": "secret-password",
    "ssl": true
  }
}

Assurez-vous de stocker vos logs. Si vous souhaitez pouvoir relancer des transactions, conservez les données du message et fournissez une interface pour manipuler ces données à la main, ou réexécuter certaines transactions après que l'expression du job ait été modifiée.

Veillez à ce qu'un seul message entrant puisse lancer l'exécution de plusieurs jobs.

Pour commencer, ou exécutez simplement fn-lang manuellement à partir de votre ligne de commande, consultez openfn-devtools. Avec des scripts d'installation pour Windows et Linux, c'est le moyen le plus rapide de mettre en place et de faire fonctionner OpenFn sur votre machine locale.

Exemple de fichiers pour démarrer DIY​

Vous trouverez ci-dessous un exemple de code pour remplir les 3 fichiers nécessaires à l'exécution de fn-lang - message.json, expression.js et config.json.

message.json​

{
  "xform_ids": [],
  "version": null,
  "user_id": "user1",
  "server_date_opened": null,
  "server_date_modified": null,
  "properties": {
    "prop_c": "2013-05-18",
    "prop_b": "Female",
    "prop_a": 99,
    "owner_id": null,
    "external_id": null,
    "date_opened": null,
    "date": "2013-05-17",
    "case_type": "case_type",
    "case_name": "Demo"
  },
  "indices": {}
}

expression.js​

event(
  fields(
    field('program', 'eBAyeGv0exc'),
    field('orgUnit', 'DiszpKrYNg8'),
    field('eventDate', dataValue('properties.date')),
    field('status', 'COMPLETED'),
    field('storedBy', 'admin'),
    field('coordinate', {
      latitude: '59.8',
      longitude: '10.9',
    }),
    field('dataValues', function (state) {
      return [
        {
          dataElement: 'qrur9Dvnyt5',
          value: dataValue('properties.prop_a')(state),
        },
        {
          dataElement: 'oZg33kd9taw',
          value: dataValue('properties.prop_b')(state),
        },
        {
          dataElement: 'msodh3rEMJa',
          value: dataValue('properties.prop_c')(state),
        },
      ];
    })
  )
);

config.json​

{
  "username": "admin",
  "password": "district",
  "apiUrl": "https://play.dhis2.org/demo"
}
