Portabilité

Intentions​

Au-delà de faciliter la portabilité / transférabilité entre la plateforme d'OpenFn et les voies de déploiement microservice, la proposition de portabilité (nécessite un meilleur nom... et...) cherche à établir un moyen simple et applicable à l'échelle mondiale pour spécifier l'automatisation du flux de travail qui pourrait être appliquée à travers les moteurs de flux de travail / plates-formes d'intégration à travers le secteur. Rien dans la spécification ne doit être spécifique à OpenFn ou à l'un de nos produits individuels. Nous envisageons un avenir dans lequel les logiciels basés sur le noyau, le moteur, et les outils d’intégration/de flux de travail entièrement nouveaux et différents peuvent adopter cette spécification.

Il se résume à trois ensembles clés d'artefacts : les taches, les déclencheurs , et les identifiants. Respectivement, ils déterminent (1) quelles actions doivent être effectuées, (2) quand elles doivent être effectuées, et (3) quelle authentification, le cas échéant, ils auront besoin pour les effectuer.

Si vous souhaitez contribuer à la spécification, contactez OpenFn via le forum de la communauté, écrivez-nous ou suggérez des modifications en soumettant une demande ici.

Proposition v2 @dernier​

tâches :
  tâche-1 :
    expression: >
      registerPatient({
        patient-id: state.data.id,
        dob: state.data.birth
      })
    adaptateur :  '@openfn/language-openmrs'
    déclencheur : déclencheur-1
    identifiant: mon-secret-identifiant
  tâche-récurrente :
    expression : >
      alterState(state => {
        console. og( « Salut ! »)
        état de retour ;
      })
    adaptateur : '@openfn/language-common'
    declencheur : chaque-minute
 flux-tâche : 
    expression : >
      alterState(state => {
        state. data.number = state.data. numéro * 3
        état de retour ;
      })
    adaptateur : '@openfn/language-common'
    identifiant : après-j1
  attrape-tâche :
    expression : >
      alterState(state => {
        state. message = « l'a traité. »
        état de retour ;
      })
    adaptateur : '@openfn/language-common'
    declencheur: j1-échec

declencheurs :
  declencheur-1 :
    critères: '{"number":2}'
  chaque-minute: 
    cron: '* * * * * *
  après-j1 :
    Succès : tâche-1
  j1-echec:
    echec: tâche-1

# Notez que les clés d'identification sont copiées, mais les valeurs doivent être saisies manuellement
# une fois l'exportation terminée.
identifiants:
  my-secret-credential:
    nom d'utilisateur : '******'
    mot de passe : '******'
